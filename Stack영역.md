# Stack 영역

프로그램이 실행되는 동안 함수 호출과 로컬 데이터 처리를 위해 관리되는 매우 역동적이고 중요한 메모리 공간

## 특징

스택 영역은 이름 그대로 LIFO(Last In First Out - 후입선출) 구조로 동작하는 메모리 공간

- **자동 메모리 관리 :** 함수가 호출되면 해당 함수만을 위한 공간이 자동으로 할당되고, 함수가 종료(리턴)되면 자동으로 메모리가 해제됨.
- **매우 빠른 접근 속도 :** CPU의 스택 포인터(Stack Pointer) 레지스터가 현재 스택의 최상단 주소를 가리키고 있기 때문에, 메모리 할당과 해제가 단순히 포인터 위치를 이동하는 것으로 끝남. 따라서 힙(Heap) 영역에 비해 속도가 압도적으로 빠름.
- **컴파일 타임에 크기 결정 :** 스택 영역의 최대 크기는 프로그램이 실행되기 전(컴파일 타임)에 운영체제나 컴파일러 설정에 의해 정해짐.
- **높은 주소에서 낮은 주소로 할당 :** 일반적으로 스택은 메모리의 높은 주소(High Address)에서 시작하여 아래(Low Address) 방향으로 자라남.
    - **스택 오버플로우 (Stack Overflow) :** 스택 영역은 크기가 제한되어 있어 설정된 스택 메모리 한계를 초과하면 프로그램이 강제 종료되는 스택 오버플로우 에러가 발생함.

## 스택 영역에 저장되는 데이터 종류

스택 영역에는 함수가 호출될 때 생성되는 스택 프레임(Stack Frame)이라는 단위 내에 다음과 같은 데이터들이 저장됨.

- **지역 변수 (Local Variables) :** 함수 내부에서 선언된 변수들. 함수가 실행되는 동안만 존재하며, 함수가 끝나면 사라짐.
- **매개변수 (Parameters / Arguments) :** 함수를 호출할 때 인자로 전달되는 값.
- **반환 주소 (Return Address) :** 함수 수행이 끝난 후, 원래 호출했던 곳(부모 함수)의 다음 명령어로 돌아가기 위한 CPU 명령어 주소
- **이전 프레임 포인터 (Base Pointer) :** 이전 함수의 스택 프레임 시작 위치를 기억하여, 현재 함수가 끝났을 때 이전 함수의 환경을 복원하는 데 사용됨

## 다른 메모리 영역과의 상호작용

### 힙(Heap) 영역과의 상호작용

- **참조(Reference) 저장 :** C#이나 Java 같은 객체 지향 언어에서 new 키워드로 생성된 실제 객체(인스턴스)는 크기가 유동적이기 때문에 힙 영역에 저장. 이때 스택 영역의 지역 변수는 힙 영역에 있는 실제 객체의 메모리 주소(참조값)만을 가짐.
- **메모리 충돌 방지 :** 힙은 낮은 주소에서 높은 주소로, 스택은 높은 주소에서 낮은 주소로 자라남. 두 영역은 같은 가상 메모리 공간을 공유하므로, 한쪽이 너무 커져서 상대 영역을 침범하지 않도록 조율됨.

### 코드(Code)/데이터(Data) 영역과의 상호작용

- **코드 영역 실행 흐름 제어 :** CPU가 코드 영역에 있는 함수 명령어들을 한 줄씩 실행하다가 함수 호출을 만나면, 스택 영역에 반환 주소를 push하고 해당 함수의 코드 위치로 점프함.
- **전역/정적 변수와의 분리 :** 데이터 영역에 저장되는 전역 변수나 정적(Static) 변수는 프로그램 시작부터 종료까지 유지되지만, 스택의 지역 변수는 함수가 실행될 때만 생성되어 데이터 영역의 변수들과 수명을 완전히 달리하며 협력함.
