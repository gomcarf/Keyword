# WAV

- WAV(Waveform Audio File Format)은 IBM과 마이크로소프트가 공동 개발한 오디오 파일 포맷

## 정의

- 디지털 오디오 데이터를 가공 없이 그대로 저장하는 대표적인 무압축, 무손실 포맷
- 아날로그 소리 신호를 일정 시간 간격으로 샘플링하고 각 샘플의 소리 크기를 디지털 값으로 양자화하여 있는 그대로 기록

## 특징

1. **무손실/고음질(Lossless Quality)** : 데이터를 압축하여 깎아내는 과정이 없기 때문에 녹음된 원음의 주파수와 다이나믹 레인지를 100% 보존
2. **낮은 시스템 부하(Low CPU Overhead)** : 파일을 재생하거나 편집할 때 컴퓨터가 압축을 푸는 연산 과정을 거치지 않음. → 오디오 프로그램에서 여러 트랙을 동시에 띄워도 무리가 없음
3. **용량의 한계** : 전통적인 WAV 포맷은 파일 구조상 최대 4G까지 지원. 고음질 장신간 녹음 시 파일이 쪼개질 수 있음

## 주요 오디오 포맷과의 비교

| 특징 | WAV | FLAC | MP3 |
| --- | --- | --- | --- |
| 압축 여부 | 무압축 | 무손실 압축 | 손실 압축 |
| 음질 손실 | 없음(원본 그대로) | 없음 | 있음(인간이 못듣는 영역 삭제) |
| 용량 크기 | 매우 큼(MP3의 약 10배) | 중간(WAV의 50~60%) | 매우 작음 |
| 편집 및 처리 | CPU연산 적음(바로 재생/편집) | 압축 해제 연산 필요 | 압축 해제 연산 필요 |
| 범용성 | 매우 높음 | 보통(지원 안하는 기기 있음) | 최고(어디서나 재생 가능) |

## 주요 사용처

- **전문적인 음악 작업 및 레코딩** : 보컬, 악기 녹음이나 믹싱/마스터링 단계에서는 반드시 WAV를 사용함
- **게임 개발 및 효과음(SFX) 제작** : Unity나 Unreal 같은 게임 엔진에서 짧은 효과음을 쓸 때 WAV가 유리함. 재생할 때 CPU연산이 거의 없어 게임 성능(프레임)에 미치는 영향이 적고 반응 속도가 빠름
- **비디오 영상 편집** : 유튜브나 영화 편집 시 오디오 싱크 밀림을 방지하고 고음질을 유지하기 위해 오디오 소스는 WAV로 작업하는 것이 기본
- **아카이빙(원본 보관)** : 역사적 기록물이나 중요한 오디오 자료를 영구 보존할 때 원본 포맷으로 사용
