# TCB

| 구분 | TCB 핵심 내용 |
| --- | --- |
| 개념 | 스레드별 정보를 담은 운영체제의 관리 대장 |
| 저장위치 | OS 커널 메모리 영역 |
| 핵심 데이터 | 스레드 ID, 스레드 상태, 스택 포인터 |
| 존재 이유 | 끊김 없는 스레드 교체와 효율적인 작업 스케줄링 |

## 정의

- Thread Control Block : OS 커널(Kernel)이 관리하는 데이터 구조
- 특정 스레드의 상태, 실행 정보, CPU 레지스터 값 등 스레드 하나를 제어하는 데 필요한 모든 정보를 담고 있는 메모리 블록
- OS는 문맥 교환이 이러날 때, 기존 스레드의 진행 상황을 TCB에 저장하고 새로운 스레드의 TCB에서 정보를 읽어와 작업을 이어감.

## 주요 기능

- 스레드 상태 및 진행 상황 보존(Context Saving) : CPU가 다른 스레드로 전환될 때, 현재 스레드가 어디까지 실행되었는지 레지스터 값을 TCP에 기록하여 나중에 정확히 그 시점부터 다시 실행할 수 있게 함.
- 스레드 스케줄링 지원 : OS의 스케줄러가 다음 실행 스레드를 결정할 수 있도록 각 스레드의 우선순위, 상태 정보를 제공
- 독립적 자원 관리 : 스레드가 개별적으로 가지는 스택 영역의 주소(Stack Pointer)와 로컬 데이터(Thread Local Storage)를 관리하여 스레드 간 작업이 엉키지 않도록 격리함.

## 특징

- PCB와의 종속 관계 : TCB는 독자적으로 존재할 수 없으며 반드시 특정 PCB내부에 포함되거나 PCB를 가리키는 포인터를 가짐
- 가벼운 문맥 교환 :  프로세스를 바꿀 때 메모리 맵까지 전체를 바꿔야 해서 무거움. 반면, 같은 프로세스 내에서 스레드를 교체할 때는 공유하는 자원을 두고 레지스터 값만 가볍게 교환하므로 오버헤드가 훨씬 적다.
- 커널 공간(Kernel Space)에 저장 : 보안과 안정성을 위해 TCB는 일반 사용자가 접근할 수 없는 OS커널의 안전한 메모리 영역에 생성되고 관리됨.

## 구성요소

| 구성요소 | 역할 및 구체적 내용 |
| --- | --- |
| Thread ID(TID) | 시스템 내에서 특정 스레드를 유일하게 식별하기 위한 고유 번호 |
| Thread State | 현재 스레드의 상태 |
| Program Counter(PC) | 이 스레드가 다음에 실행해야 할 기계어 명령어의 주소 |
| Register Set | CPU의 일반 레지스터, 누산기 등의 현재 값(문맥교환 시 보존됨) |
| Stack Pointer(SP) | 스레드만의 독립적인 메모리 공간인 스택의 최상단 주소 |
| Priority | 스케줄러가 CPU를 할당할 때 참고하는 스레드의 우선순위 |
| Parent PCB Pointer | 이 스레드가 소속되어 있는 부모 프로세스(PCB)의 주소 정보 |
