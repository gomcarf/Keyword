# PCB - Process ID

| **구분** | **항목** | **주요 내용** |
| --- | --- | --- |
| **주요 기능** | **고유 식별** | 똑같은 프로그램이 여러 개 켜져 있어도 각각을 개별 개체로 구분 |
|  | **시스템 호출 대상** | 특정 프로세스를 종료(`kill`)하거나 상태를 제어할 때 타겟으로 사용 |
|  | **계층 구조 관리** | 부모 PID(PPID)와 연계하여 프로세스 간의 족보(부모-자식 관계) 형성 |
|  | **자원/권한 관리** | 어떤 프로세스가 자원(파일, 메모리)에 접근할 권한이 있는지 검증 |
| **핵심 특징** | **유일성** | 실행 중인 프로세스 간에는 절대 번호가 중복되지 않음 |
|  | **제한된 범위** | 시스템 OS에 따라 가질 수 있는 최대 번호(`PID_MAX`)가 정해져 있음 |
|  | **재사용성** | 프로세스 종료 시 번호는 반환되지만, 혼선을 막기 위해 즉시 재할당되지 않음 |
|  | **순차적 할당** | 일반적으로 새로 생성될 때마다 번호가 1씩 증가하며 부여됨 |


- 운영체제에서 PCB(Process Control Block, 프로세스 제어 블록)는 커널이 프로세스를 관리하기 위해 필요한 모든 정보를 담고 있는 거대한 데이터 구조체.

- 모든 프로세스에는 프로세스의 식별자를 저정하는 고유의 id가 할당되며 이를 Process ID (PID, 프로세스 식별자)라고 함.

## 1. Process ID (PID)의 주요 기능

### ① 프로세스의 고유한 식별 (Identification)

- **역할 :** 시스템 내에서 실행 중인 수많은 프로세스 중 특정 프로세스를 유일하게 가리키기 위해 사용.
- **중요성 :** 메모리에 똑같은 프로그램(예: 크롬 브라우저 창 3개)이 여러 개 실행되어 있어도, 운영체제는 각각 부여된 서로 다른 PID를 보고 이들을 개별적인 개체로 구분.

### ② 시스템 호출(System Call)의 대상 지정

- **역할 :** 특정 프로세스를 제어하기 위한 시스템 호출을 보낼 때 대상 지정자로 사용됨.
- **예시 :**
    - 프로세스를 강제로 종료하고 싶을 때: `kill -9 <PID>` 명령어를 통해 해당 PID를 가진 프로세스에만 종료 신호(Signal)를 보냄.
    - 특정 프로세스의 리소스 사용량을 조회하거나 우선순위를 바꿀 때도 PID가 기준이 됨.

### ③ 프로세스 간 관계 정의 및 계층 구조 관리

- **역할 :** 운영체제는 프로세스 간의 부모-자식 관계를 관리합니다. 이때 PID가 핵심 고리로 작동.
- **부모 프로세스 ID(PPID, Parent PID) :** PCB 내에는 자신의 PID뿐만 아니라 자신을 생성해 준 부모 프로세스의 PID(PPID)도 함께 저장되어 있어, 프로세스들의 계층적 족보(Tree Structure)를 구성할 수 있게 함.

### ④ 자원 및 권한 관리 (Access Control)

- **역할 :** 특정 프로세스가 파일, 네트워크 소켓, 메모리 영역 등의 시스템 자원에 접근할 권한이 있는지 검사할 때 PID를 참조.
- **사용자 ID(UID)/그룹 ID(GID)와의 연계 :** 특정 PID가 어떤 사용자의 권한으로 실행되었는지 매핑하여 보안 및 접근 제어를 수행.

## 2. PID의 핵심적인 특징

| **특징** | **설명** |
| --- | --- |
| **유일성 (Uniqueness)** | 임의의 한 시점에 실행 중인 모든 프로세스 사이에서 PID는 절대 중복될 수 없음. |
| **제한된 범위 (Finite Range)** | PID는 무한한 숫자가 아닙니다. 시스템(OS)에 따라 최대치(`PID_MAX`)가 정해져 있음. (예: 64비트 리눅스의 경우 기본값은 대략 32,768 시스템 콜 설정을 통해 더 크게 늘릴 수 있음) |
| **재사용성 (Reusability)** | 프로세스가 종료되면 사용하던 PID는 운영체제에 반환됨. 하지만 시스템은 방금 종료된 PID를 즉시 다른 프로세스에 할당하지 않고, 한 바퀴 순환하거나 일정 시간이 지난 후에 재사용(지연 재사용)하여 혹시 모를 혼선을 방지. |
| **생성 규칙 (Sequential Allocation)** | 일반적으로 프로세스가 새로 생성될 때마다 마지막으로 할당된 PID에서 1씩 증가하는 순차적 방식으로 부여됨. 단, 최대치에 도달하면 다시 사용 가능한 작은 번호부터 찾아서 할당. |
