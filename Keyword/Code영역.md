# Code 영역

- 실행할 프로그램의 기계어 명령(Instruction)이 저장되는 메모리 공간
- Text 영역이라고도 부름

## 특징

- **읽기 전용(Read-Only)** : 프로그램이 실행되는 동안 코드의 내용이 변경되는 것을 방지하기 위해 쓰기(Write) 권한이 금지되어 있음.
- **생명 주기** : 프로그램이 시작될 때 메모리에 할당되고, 프로그램이 완전히 종료될 때까지 유지 됨.
- **고정된 크기(Static Size)** : 컴파일 타임(프로그램이 빌드되는 시점)에 이미 필요한 크기가 결정됨. 프로그램 실행 중에 크기나 늘어나거나 줄어들지 않음.

## 코드 영역에 저장되는 내용

- **컴파일 된 기계어 명령(CPU Instructions)** : if문, for문, 산술 연산 등을 수행하는 실제 CPU 명령어들
- **함수(Function)의 본체** : 제어 흐름과 로직을 담당하는 함수의 실제 실행 코드
- **코드 내의 리터럴 상수** : 소스 코드에 직접 입력한 문자열 상수 등 변경되어서는 안되는 데이터들 (컴파일러나 OS 환경에 따라 Data 영역에 저장되기도 함)

## 다른 영역과의 상호작용

### CPU와의 상호작용

CPU 안에는 다음에 실행할 명령어의 메모리 주소를 가리키는 PC(Program Counter)라는 레지스터가 존재함

1. CPU는 PC가 가리키는 코드 영역의 주소에서 명령어를 하나 씩 가져옴(Fetch)
2. 가져온 명령이 무엇인지 해석(Decode)
3. 명령을 실행(Execute)
4. 함수 호출이나 제어문(if, loop)을 만나면 코드 영역 내의 다른 주소로 점프하며 과정을 반복.

### 스택(Stack) 영역과 상호작용

함수가 실행되는 과정에서 두 영역은 가장 빈번하게 상호작용

- 코드 영역에서 “A 함수를 실행하라”는 명령어가 실행되면, 스택 영역에서 A함수를 위한 공간인 스택 프레임이 만들어짐
- 이때 코드 영역의 다음 실행해야 할 메모리 주소를 스택에 복사해 둠. 함수가 끝나면 스택에 저장된 이 주소를 보고 다시 코드 영역의 원래 위치로 돌아가 다음 명령을 이어감

### 데이터(Data) 영역과 상호작용

프로그램 전체에서 공유하는 전역 변수나 정적 변수가 저장되는 곳과의 상호작용

- 코드 영역의 명령어가 실행될 때, CPU는 데이터 영역의 특정 메모리 주소로 찾아가 값을 읽어오거나 수정
- 코드 영역은 읽기 전용이지만 명령을 통해 데이터 영역의 값은 자유롭게 읽고 쓰기가 이루어짐

### 힙(Heap) 영역과 상호작용

런타임에 프로그래머가 필요에 따라 메모리를 할당

- 코드 영역에 new 명령어가 있으면 시스템은 힙 영역에서 빈 공간을 찾아 할당
- 힙 영역에 생성된 실제 데이터의 메모리 주소는 스택 영역에 저장됨. 코드 영역의 명령어들은 스택에 있는 포인터를 거쳐 힙 영역의 데이터를 조작함.
