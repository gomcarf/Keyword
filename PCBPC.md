# PCB - PC

| **구분** | **PCB 내부의 Program Counter** | **CPU Register의 Program Counter** |
| --- | --- | --- |
| **개념적 본질** | 프로세스의 상태를 보관하는 소프트웨어적 백업 구조체 | 명령어를 순차적으로 실행하는 하드웨어 제어 장치 |
| **물리적 위치** | 메인 메모리(RAM)의 커널(Kernel) 영역 | CPU(프로세서) 내부 레지스터 |
| **작동 제어 주체** | 운영체제(OS) 스케줄러 및 커널 소프트웨어 | CPU 내부의 제어 장치(Control Unit) 하드웨어 |
| **값의 변화** | 프로세스가 CPU에서 내려올 때 단 한 번 기록 및 고정 | 명령어가 실행될 때마다(매 클럭마다) 실시간으로 증가/변경 |
| **존재 목적** | 멀티태스킹 시 이전 실행 상태를 안전하게 복구하기 위함 | 현재 할당된 명령어를 끊임없이 수행하기 위함 |

## 정의

- PCB 내의 Program Counter는 특정 프로세스가 CPU 할당을 사용하고 대기 상태로 전환될 때, 다음번에 돌아와서 실행해야 할 명령어의 메모리 주소를 보관하는 저장 공간(소프트웨어적 변수).

## 기능

- **문맥 교환(Context Switching) 지원 :** CPU가 A라는 프로그램을 구동하다가 B라는 프로그램을 처리하기 위해 이동할 때, 프로그램 A가 어디까지 실행되었는지 주소를 PCB의 PC에 백업.
- **프로세스 이어 하기 :** 다시 프로그램 A의 차례가 오면, 운영체제는 PCB에 저장해 두었던 PC 값을 읽어와 CPU에 넣어줌. 덕분에 프로그램 A는 멈췄던 부분부터 정확히 다시 시작됨.

## 특징

- **RAM(메인 메모리)에 존재 :** CPU 내부가 아니라, 커널이 관리하는 메인 메모리 공간에 위치.
- **대기 상태일 때만 유효함 :** 해당 프로세스가 CPU를 차지하고 실행되는 중에는 이 값이 업데이트 되지 않고 가만히 있으며, CPU에서 없어지는 순간의 주소가 기록됨.

## CPU Register의 Program Counter와의 차이점

- **CPU Register의 PC :** 하드웨어 칩 내부에 존재하는 초고속 메모리. 현재 CPU가 **"**지금 당장 실행하고 있는**"** 명령어 주소를 가리키며, 1GHz 컴퓨터라면 1초에 10억 번씩 값이 실시간으로 바뀜.
- **PCB의 PC :** 메인 메모리에 구조체 형태로 존재하는 기록부. 현재 CPU를 쓰지 못하고 대기실에서 기다리는 프로세스들의 "과거 마지막 실행 위치"를 붙잡고 있는 정적인 백업 데이터.
