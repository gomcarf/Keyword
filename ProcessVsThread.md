# Process vs Thread

| **구분** | **프로세스 (Process)** | **스레드 (Thread)** |
| --- | --- | --- |
| **개념** | 실행 중인 프로그램 (독립된 단위) | 프로세스 내의 실행 흐름 (작업 단위) |
| **메모리** | 독립적으로 할당 (Code, Data, Stack, Heap 모두 독립) | Stack만 독립, Code/Data/Heap은 공유 |
| **통신 비용** | 높음 (IPC 필요) | 낮음 (공유 메모리 바로 접근 가능) |
| **전환 속도** | 느림 (Context Switching 비용 큼) | 빠름 (Context Switching 비용 적음) |
| **장애 영향** | 다른 프로세스에 영향 없음 | 하나의 스레드가 죽으면 프로세스 전체가 죽음 |

## 1. 프로세스 (Process)

컴퓨터에서 메모리에 적재되어 CPU에 의해 실행되고 있는 상태의 프로그램

- **독립성 :** 프로세스는 OS로부터 독립된 메모리 영역(Code, Data, Stack, Heap)을 할당받음. 다른 프로세스의 메모리 영역에 직접 접근 불가.
- **안전성 :** 한 프로세스가 잘못되어 정지하더라도 다른 프로세스에는 영향을 주지 않음. (예: 크롬 브라우저가 멈춰도 카카오톡은 잘 작동함)
- **비용(Overhead) :** 프로세스를 새로 생성하거나, 프로세스 간에 전환(Context Switching)할 때 많은 시간과 메모리가 소모됨.
- **통신(IPC) :** 독립된 구조 탓에 프로세스들끼리 데이터를 주고받으려면 별도의 통신 메커니즘(IPC: Inter-Process Communication, 예: 파이프, 소켓, 공유 메모리 등)이 필요.

---

## 2. 스레드 (Thread)

하나의 프로세스(회사) 안에서 실제로 작업을 수행하는 주체

- **자원 공유 :** 스레드들은 부모 프로세스의 메모리 영역 중 Code, Data, Heap 영역을 서로 공유. 단, 각 스레드는 독립적인 실행 흐름을 가져야 하므로 Stack 영역과 CPU 레지스터는 개별적으로 할당 받음.
- **효율성 :** 메모리를 공유하기 때문에 스레드 간의 통신이 매우 빠르고, 자원 생성 및 전환(Context Switching) 비용이 프로세스에 비해 훨씬 적다. (이 때문에 스레드를 경량 프로세스(Lightweight Process)라고도 부릅니다.)
- **취약성 :** 자원을 공유한다는 것은 양날의 검. 하나의 스레드가 잘못되어 프로세스 자체를 강제 종료시키면, 그 안의 모든 스레드가 함께 종료됨. 또한, 동시에 같은 데이터에 접근할 때 데이터가 꼬이는 문제(동기화 문제)가 발생할 수 있어 주의가 필요.
