# Data 영역

- 프로그램의 전역적인 상태를 관리하는 핵심 공간

## 특징

- 프로그램이 시작될 때 할당되어 프로그램이 종료될 때까지 메모리에 유지되는 영역
- **고정된 크기** : 컴파일 단계에서 필요한 메모리 크기가 결정되며, 실행 중에 크기가 늘어나거나 줄어들지 않음.
- **전역적 접근 가능** : 프로그램 내부 어디서든(어떤 함수에서든) 이 영역에 저장된 데이터에 접근 가능
- **느린 초기화와 빠른 접근** : 프로그램 시작 시점에 메모리에 로드 되므로 초기화 비용이 들지만, 실행 중에는 주소 값이 고정되어 있어 접근 속도가 매우 빠름

## 데이터 영역에 저장되는 내용

데이터 영역은 효율적인 메모리 관리를 위해 내부적으로 크게 두 가지 영역으로 다시 세분화 됨.

1. **.data 영역 (Initialized Data Segment)**
    - 내용 : 초기화된 전역 변수와 초기화된 정적(static) 변수가 저장됨
    - 특징 : 프로그램 소스 코드에서 개발자가 명시적으로 값을 지정한 변수들. 이 값들은 ROM/플래시 메모리 등에 저장되어 있다가 프로그램 실행 시 RAM의 .data 영역으로 복사 됨.
2. **.bss 영역 (Block Started by Symbol)**
    - 내용 : 초기화되지 않은 전역 변수와 초기화되지 않은 정적 변수가 저장됨
    - 특징 : 프로그램이 실행될 때 OS에 의해 모두 0(또는 NULL)으로 자동 초기화. 소스 코드에 변수 이름과 타입만 있고 값이 없기 때문에, 컴파일된 바이너리 파일의 크기를 줄이기 위해 별도의 공간으로 분리하여 관리함

## 다른 메모리 영역과 상호작용

```csharp
+-----------------------------------+ High Memory Addresses
|        스택 영역 (Stack)           |  (지역 변수, 매개변수 / 아래로 자람)
+-----------------------------------+
|                 ↓                 |
|        힙 영역 (Heap)             |  (동적 할당 / 위로 자람)
+-----------------------------------+
|       데이터 영역 (Data)           |  (전역, 정적 변수 / .data 및 .bss)
+-----------------------------------+
|        코드 영역 (Code)           |  (CPU 기계어 명령어)
+-----------------------------------+ Low Memory Addresses
```

### 코드 영역(Code)과의 상호작용

- CPU가 코드 영역의 명령을 수행하다가 전역 변수와 관련된 명령(ex: 전역변수에 5를 더하라)을 만나면, 코드 영역의 명령어는 **데이터 영역에 있는 특정 메모리 주소를 참조**하여 값을 읽어오고 수정.

### 스택 영역(Stack)과의 상호작용

- 함수가 호출되면 스택 영역에 해당 함수의 지역 변수들이 쌓임.
- 이때 함수 내부에서 전역 변수(데이터 영역)의 값을 가져와 지역 변수(스택 영역)에 대입하거나 연산할 수 있음.
- 반대로, 데이터 영역에 있는 **전역 포인터 변수**가 스택 영역에 있는 지역 변수의 주소를 가리키도록 설정할 수도 있음. (다만 함수가 종료되면 스택의 변수는 소멸하므로 주의)

### 힙 영역(Heap)과의 상호작용

- 힙 영역은 개발자가 실행 중에 할당하는 동적 메모리 공간(C의 malloc, C++의 new)
- 전역 변수로 선언된 포인터(데이터 영역)가 힙 영역에 생성된 동적 객체의 주소를 가리키는 방식으로 자주 사용됨.
- 이 방식을 사용하면, 프로그램 전체에서 언제든 데이터 영역의 포인터를 통해 힙 영역에 할당된 거대한 데이터 구조(예: 싱글톤 객체, 전역 매니저 클래스 등)에 접근할 수 있음.

## 짚고 넘어가기: 전역 변수 남용을 지양하는 이유

데이터 영역은 프로그램 종료 시까지 메모리를 독점하므로, 너무 많은 전역/정적 변수를 선언하면 메모리 낭비가 심해짐. 또한 어떤 함수에서든 접근하여 수정할 수 있기 때문에 데이터 영역의 값이 꼬이면 버그를 추적하기 매우 어려워짐. 따라서 꼭 필요한 경우가 아니라면 스택(지역 변수)이나 힙(동적 할당)을 적절히 조합하여 사용하는 것이 좋다.
