# 파일 포맷(File Format)

## 정의

- 컴퓨터에서 데이터를 저장하고 식별하는 표준화된 규칙(구조)

## 주요 기능

- **데이터의 구조화 및 표준화** : 바이너리로 이루어진 컴퓨터 데이터를 규칙에 맞게 배치하여 프로그램이 올바르게 읽고 쓸 수 있는 틀을 제공
- **컴퓨터와 프로그램 간의 상호작용(호환성)** : 운영체제나 소프트웨어가 파일의 확장자를 보고 어떤 프로그램으로 이 파일을 열어야 하는지 즉시 판단하고 연결
- **데이터 압축 및 최적화** : 파일의 목적에 맞게 용량을 줄이거나 화질을 높이는 등 데이터를 최적의 상태로 저장

## 설계 목적에 따른 특징

| 특징 종류 | 설명 | 대표적인 예시 |
| --- | --- | --- |
| 압축 방식(Compression) | 데이터를 저장할 때 용량을 줄이는 방식. 화질/ 음질 손실을 감수하고 용량을 대폭 줄이는 손실 압축과 원본을 그대로 보존하는 무손실 압축으로 나뉨 |   • 손실: JPEG, MP3
  • 무손실: PNG, FLAC |
| 범용성(Compatibility) | 특정 프로그램에서만 열리는지, 혹은 모든 기기에서 제한 없이 열리는지 여부 |   • 범용성 높음 : TXT, PDF, MP4
  • 독점적 포맷 : PSD |
| 메타데이터 포함(Metadata) | 파일 본문 데이터 외에 파일에 대한 부가 정보를 함꼐 저장하는 특징 |   • 사진의 Exif 정보(JPEG) |
| 이진 vs 텍스트 | 사람이 메모장으로 읽을 수 있는 형태(텍스트)인지, 컴퓨터만 해석할 수 있는 이진 코드 형태인지 |   • 텍스트 : HTML, JSON, XML
  • 바이너리 : EXE, PNG, ZIP |

## 파일 확장자(File Extension)

- 컴퓨터는 파일의 종류를 쉽게 구별하기 위해 파일 이름 맨 뒤에 점과 함께 3~4글자의 확장자를 붙여 포맷을 나타냄

### 대표적인 파일 포맷 종류

#### 1. 텍스트 / 문서 포맷

- .txt : 가장 기본적인 텍스트 포맷으로, 서식없이 글자만 저장. 용량이 매우 작고 어떤 기기에서나 열림
- .docx / .hwp : 마이크로소프트 워드와 아래아 한글의 포맷. 글자 뿐만 아니라 글자 크기, 색상, 표, 이미지 등 복잡한 서식을 함께 저장
- .pdf : 어도비에서 개발한 포맷으로, 어느 기기나 프로그램에서 열어도 서식과 글꼴이 깨지지 않고 똑같이 보이도록 고정하는데 특화

#### 2. 이미지 포맷

- .jpg / .jpeg : 압축률이 높아 용량이 작으면서도 화질이 좋아 웹이나 스마트폰 사진에 가장 많이 쓰임(단, 압축할 때 화질 손실이 약간 있음)
- .png : 화질 손실이 없고 배경을 투명하게 처리할 수 있어서 로고나 웹그래픽에 필수적
- .gif : 움직이는 이미지와 간단한 애니메이션을 지원하는 포맷

#### 3. 오디오 / 동영상 포맷

- .mp3 : 인간이 듣지 못하는 주파수 영역을 지워 용량을 획기적으로 줄인 가장 대중적인 음악 포맷
- .mp4 : 비디오와 오디오를 모두 담는 가장 표준적인 동영상 포맷으로, 호환성이 매우 뛰어남
