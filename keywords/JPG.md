# JPG

## 정의

- 디지털 이미지를 저장하는 데 가장 널리 쓰이는 대표적인 파일 포맷.
- 웹 서핑, 스마트폰 사진, SNS등 일상에서 접하는 대부분의 사진 파일의 형식

## JPG의 압축 방법

#### 1단계: 사람의 눈을 속이는 변환 (Color Space Conversion)

- 컴퓨터는 보통 빛의 삼원색인 RGB(Red, Green, Blue)로 색을 표현함. 하지만 JPG는 이 상태로 압축하지 않고, $YC_bC_r$라는 색상 공간으로 변환.
    - $Y$ (Luminance, 휘도) : 이미지의 밝기 정보 (흑백 사진이라고 보시면 됩니다.)
    - $C_b, C_r$ (Chrominance, 색차) : 이미지의 색상 정보 (파란색 색차, 빨간색 색차)
- 핵심 원리 : 사람의 눈은 밝기 변화($Y$)에는 매우 민감하지만, 색상 변화($C_b, C_r$)에는 엄청나게 둔감. (예시: 어두운 방에서 물체의 형태(밝기)는 알아봐도 정확한 색은 구별하기 힘듦)

#### 2단계: 색상 정보만 반으로 버리기 (Chroma Subsampling)

- 눈에 민감한 밝기($Y$) 정보는 그대로 두고, 둔감한 색상($C_b, C_r$) 정보만 해상도를 강제로 낮추어 버림.
- 가장 흔히 쓰이는 4:2:0 다운샘플링 방식 : 가로 2픽셀, 세로 2픽셀 총 4개의 픽셀이 있다고 가정.
    - 밝기($Y$) 정보는 4개 픽셀 모두 각각 따로 저장
    - 색상($C_b, C_r$) 정보는 4개 픽셀이 그냥 1개의 공통된 색상 값을 나눠 갖도록 뭉뚱그림

⇒ 이 단계까지만 와도 데이터 용량이 원래의 절반 가까이 줄어들지만, 사람은 눈치채지 못함.

#### 3단계: 이미지를 파형(주파수)으로 바꾸기 (DCT, Discrete Cosine Transform)

- 이제 이미지를 $8 \times 8$ 크기의 작은 블록(총 64개 픽셀)으로 쪼개고 이 블록에 이산 코사인 변환(DCT)이라는 수학적 필터를 적용.
- 픽셀의 색상 값을 '주파수(Frequency)' 성분으로 변환하는 작업.
- 변환을 하고 나면 $8 \times 8$ 블록의 왼쪽 위에는 전체적인 거대한 색상 흐름(저주파)이 모이고, 오른쪽 아래에는 미세하고 복잡한 디테일 변화(고주파)가 모이게 됨.

#### 4단계: Quantization, 양자화

- 실질적으로 화질 저하와 용량 감소가 일어나는 단계
- DCT를 거쳐 주파수별로 정렬된 $8 \times 8$ 표를 '양자화 테이블(Quantization Table)'이라는 특정 숫자가 적힌 표로 나눔.
    - 저주파(중요한 전체 색상) 영역: 작은 숫자로 나누기 때문에 원래 값이 잘 보존.
    - 고주파(미세한 디테일) 영역: 매우 큰 숫자로 나눔.
        - 예시: 미세한 디테일 값이 13, 25 같은 숫자가 있었는데 이를 큰 숫자인 50으로 나누고 소수점을 반올림하면 모두 0이 됨.
- $8 \times 8$ 표에서 오른쪽 아래에 있던 미세한 색상 변화 데이터들이 전부 0으로 초기화되면서 사라집니다.

#### !주의!

- 이렇게 수많은 데이터가 0으로 바뀌면, 컴퓨터는 데이터를 엄청나게 압축할 수 있게 됨.
- JPG 사진을 크게 확대했을 때 바둑판 모양으로 깨지는 현상(블록 현상)이 생기는 이유가 바로 이 $8 \times 8$ 블록 단위로 색상을 너무 심하게 뭉뚱그렸기 때문.

## 특징

1. **손실 압축(Lossy Compression)**
    - 디테일한 색상 정보를 미세하게 지우고 비슷한 색상끼리 뭉뚱그려서 파일 크기를 획기적으로 줄임
    - 장점 : 파일 용량이 매우 작아서 저장 공간을 아낄 수 있고 인터넷에서 이미지를 주고받거나 웹페이지를 로딩할 때 속도가 매우 빠름
    - 단점 : 압축을 심하게 할 수록 화질 저하(깨짐 현상, 노이즈)가 눈에 띄며, 이미지를 수정하고 다시 JPG로 저장하는 과정을 반복하면 화질이 계속 상하게 됨.
2. **JPG와 JPEG의 차이**
    - JPEG는 Joint Photographic Experts Group 의 줄임말
    - 과거 MS-DOS 시절에는 파일 확장자를 무조건 세 글자로 제한했기에 .jpg로 줄여쓰기 시작했으나 현재는 시스템 제한이 없기 때문에 모두 사용

## 장단점

- 장점
    - **높은 호환성** : 스마트폰, PC, 카메라, 웹브라우저 등 전 세계 거의 모든 기기와 소프트웨어에서 지원
    - **작은 용량** : 고화질 사진도 적절한 압축을 통해 용량을 크게 줄일 수 있음
    - **자연스러운 색상 표현** : 약 1670만 색상(24비트 컬러)을 지원하며 풍경 사진이나 인물 사진 처럼 부드럽고 복잡한 색상 변화를 표현하는 데 최적화
- 단점
    - **투명 배경 불가** : 이미지의 배경을 투명하게 만드는 기능(알파 채널)을 지원하지 않음
    - **텍스트/선 표현에 약함** : 글자나 깔끔한 그래픽, 로고 같은 이미지를 JPG로 저장하면, 경계선 주변이 찌글찌글하게 깨지는 현상(아티팩트)이 발생하기 쉬움
