# BMP

## 정의

- BMP(Bitmap Photo/Digital Image) 파일 포맷은 마이크로소프트와 IBM이 개발한 그래픽 파일 형식
- 디지털 이미지를 압축하지 않거나 단순하게 압축하여 격자 형태의 픽셀(점) 배열로 저장하는 대표적인 래스터(Raster) 그래픽 포맷
- 구조가 매우 직관적이고 단순하여 이미지 처리 프로그래밍을 처음 배울 때 자주 사용

## BMP 파일의 내부 구조

| **영역 이름** | **주요 역할 및 포함 정보** |
| --- | --- |
| **비트맵 파일 헤더**<br>(Bitmap File Header) | 파일 전체에 대한 정보 (크기: 14바이트)<br>• **파일 시그니처:** BMP 파일임을 나타내는 'BM' (0x42, 0x4D)<br>• **파일 크기:** 파일의 총 바이트 수<br>• **데이터 오프셋:** 실제 픽셀 데이터가 시작되는 위치(주소) |0
| **비트맵 정보 헤더**<br>(Bitmap Info Header / DIB) | 이미지 자체의 상세 속성 정보 (일반적으로 40바이트)<br>• **이미지 가로/세로 크기** (픽셀 단위)<br>• **색상 깊이(BPP) :** 픽셀당 비트 수 (1, 4, 8, 16, 24, 32비트)<br>• **압축 방식 :** 무압축, RLE 방식 등<br>• **이미지 데이터 크기** |
| **컬러 테이블 / 팔레트**<br>(Color Table) | 색상 인덱스 매핑 정보 (선택 사항)<br>• 1, 4, 8비트 같이 색상 수가 제한된 인덱스드 컬러(Indexed Color) 모드에서만 사용됨.<br>• 24비트(True Color) 이상의 이미지에서는 이 영역이 생략됨. |
| **픽셀 데이터**<br>(Pixel Data) | 실제 이미지를 구성하는 픽셀들의 색상 값<br>• 기본적으로 **하단에서 상단(Bottom-up)**, **좌측에서 우측** 순서로 저장됨 (이미지가 거꾸로 뒤집혀 저장되는 구조).<br>• 각 행(Row)의 데이터 크기는 반드시 **4바이트의 배수**로 패딩(Padding)되어야 함. |

## 주요 특징

- **화질 저하 없음** : 대부분 압축을 하지 않기 때문에 원본 이미지의 모든 픽셀 정보가 100% 보존됨
- **단순한 구조 :** 포맷 구조가 매우 단순하여 별도의 복잡한 라이브러리 없이도 코드로 직접 읽고 쓰기(Parsing)가 쉬움
- **빠른 처리 속도:** 이미지를 디코딩(압축 해제)하는 연산 과정이 거의 필요 없기 때문에 화면에 띄우는 속도가 빠름
- **거대한 파일 크기:** 압축을 하지 않거나 손실 없는 단순 압축만 지원하므로, JPEG나 PNG 같은 최신 포맷에 비해 파일 용량이 무지막지하게 큼
- **웹/네트워크에 부적합:** 대용량 문제로 인해 웹페이지 로딩 속도를 떨어뜨리므로 인터넷 환경에서는 거의 사용되지 않음

## 색상 깊이(BPP)별 분류

- **1비트 (흑백):** 0(검은색)과 1(흰색) 두 가지 색상만 표현.
- **8비트 (인덱스 컬러):** 최대 256가지 색상 표현. 팔레트(Color Table)를 참조하여 픽셀을 표현함. (그레이스케일 이미지에 자주 쓰임)
- **24비트 (트루 컬러):** R(레드), G(그린), B(블루)를 각각 8비트(0~255)씩 사용하여 총 16,777,216가지 색상 표현. 가장 흔하게 쓰이는 BMP 형식. (단, 파일 내 저장 순서는 RGB가 아니라 BGR 순서입니다.)
- **32비트 (알파 채널 포함):** RGB(24비트)에 투명도를 나타내는 Alpha(8비트) 채널이 추가된 형태.

## 4바이트 정렬(Padding) 규칙

- BMP 포맷을 프로그래밍할 때 가장 주의해야 하는 독특한 규칙
- 컴퓨터가 메모리를 효율적으로 읽을 수 있도록, BMP는 이미지의 가로 한 줄(행)의 바이트 크기가 무조건 4의 배수가 되도록 맞추어야 함.
- 만약 가로 픽셀 데이터의 실제 크기가 4의 배수가 아니라면, 남는 공간을 의미 없는 데이터(`0x00`)로 채워 넣는데 이를 패딩(Padding)이라고 함. 이 규칙을 맞추지 않으면 이미지가 옆으로 밀리거나 깨져 보이게 됨.
