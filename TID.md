# TCB - Thread ID

| **구분** | **주요 내용** |
| --- | --- |
| **정의** | OS가 개별 스레드를 식별하기 위해 부여하는 고유 번호 |
| **고유성 범위** | - 커널 수준: OS 전체에서 유일 (예: Linux `gettid()`)<br>- 사용자 수준: 프로세스 내부에서만 유일 (예: `pthread_self()`) |
| **핵심 기능** | ① CPU 스케줄링 대상 식별<br>② 동기화(Lock) 소유권 확인<br>③ TLS(Thread Local Storage) 주소 매핑<br>④ 디버깅 및 자원 모니터링 |
| **수명 주기** | 스레드가 생성될 때 할당되고, 종료되면 OS로 회수되어 재사용 가능 |
| **구현 형태** | - Windows: `DWORD` (32비트 부호 없는 정수) <br>- Linux: `pid_t` (정수형)<br>- POSIX (`pthread`): `pthread_t` (정수 또는 TCB를 가리키는 포인터) |
| **관계성** | 항상 부모 프로세스의 PID(Process ID) 하위에 종속됨 |

## TCB와 Thread ID의 개념

- **TCB (Thread Control Block) :** 운영체제 커널이 하나의 스레드를 관리하기 위해 필요한 모든 정보를 담고 있는 커널 내의 데이터 구조체.
- **Thread ID :** 이 TCB 내에 저장되는 가장 기본적이고 핵심적인 필드 중 하나로, 시스템 전체(또는 프로세스 내부)에서 해당 스레드를 유일하게 식별하기 위해 부여되는 고유한 번호(ID). 프로세스의 PID(Process ID)와 유사하게, 스레드 관점에서의 주민등록번호와 같은 역할.

## 주요 기능

- **스레드 식별 및 탐색 :** 커널이 특정 스레드를 대상으로 연산(스케줄링, 일시 중지, 재개, 소멸 등)을 수행할 때, 수많은 TCB 중에서 대상 스레드를 정확하게 찾아내는 인덱스 역할.
- **스레드 간 동기화 및 통신 :** 스레드 간에 데이터를 주고받거나, 특정 스레드가 다른 스레드의 종료를 대기(`join` 함수 등)할 때, 대상 스레드를 지정하는 식별자로 사용됨.
- **자원 소유권 검증 :** 뮤텍스(Mutex)나 락(Lock)을 획득할 때, 현재 어떤 Thread ID를 가진 스레드가 임계 구역(Critical Section)을 차지하고 있는지 기록하여 교착 상태(Deadlock) 방지 및 소유권 확인을 처리.
- **디버깅 및 로깅 :** 멀티스레드 애플리케이션에서 로그를 출력할 때 Thread ID를 함께 남겨, 어떤 스레드가 에러를 발생시켰는지 혹은 어떤 흐름으로 동작했는지 추적할 수 있게 함.

## 핵심 특징

- **고유성 (Uniqueness) :**
    - 활성화된 스레드들 사이에서 Thread ID는 절대 중복되지 않는다.
    - 다만, 스레드가 종료되어 자원이 반환되면 OS에 따라 해당 Thread ID가 다른 신규 스레드에 재사용될 수 있음.
- **범위 (Scope)의 차이 :**
    - **커널 수준 Thread ID (TID) :** 운영체제 커널이 직접 관리하는 ID로, 시스템 전체에서 고유함. (예: Linux의 LWP ID)
    - **사용자 수준 Thread ID :** 스레드 라이브러리(POSIX Threads의 `pthread_t` 등)가 프로세스 내부에서 관리하는 ID로, 해당 프로세스 내에서만 고유함.
- **프로세스 ID(PID)와의 관계 :**
    - 멀티스레드 환경에서 같은 프로세스에 속한 스레드들은 동일한 PID를 공유하지만, Thread ID는 각자 서로 다른 고유값을 가짐. (즉, PID가 가문의 성씨라면, Thread ID는 개인의 이름과 같다.)
