# TGA

## TGA 파일 포맷의 정의와 개념

- TARGA(Truevision Advanced Raster Graphics Adapter)의 약자
- 1984년 트루비전(Truevision)사가 자사의 그래픽 카드를 위해 개발한 비트맵(래스터) 이미지 파일 포맷
- 확장자는 .tga를 사용
- 초기 컴퓨터 그래픽 역사에서 고해상도와 진정한 색상(True Color)을 표현할 수 있는 혁신적인 포맷으로 등장
- 게임 개발, 3D 그래픽 애니메이션, 영상 편집 분야에서 텍스처 및 채널 분리용으로 널리 사용

## TGA의 핵심 특징

### 1. 알파 채널(\alpha-channel)의 완벽한 지원

- TGA 포맷은 이미지의 색상 정보(RGB) 외에도 투명도를 조절하는 '알파 채널'을 별도의 레이어로 저장
- 이미지에서 특정 부분을 투명하게 만들어 배경 위에 얹어야 하는 게임 UI, 캐릭터 텍스처, 방송용 자막 등을 제작할 때 최적의 성능을 발휘

### 2. 무손실 압축 및 비압축 방식

- 비압축(Uncompressed) : 압축을 전혀 하지 않고 이미지의 픽셀 정보를 그대로 저장. 화질 저하가 전혀 없는 대신 용량이 큽니다.
- RLE 압축(Run-Length Encoding) : 무손실 압축 알고리즘인 RLE 방식을 지원. 화질 손상 없이 파일 용량을 줄여주며, 특히 동일한 색상이 가로로 연속되는 이미지(예: 단색 배경, 단순한 아이콘 등)에서 압축 효율이 매우 높음.

### 3. 높은 호환성과 직관적인 구조

- 포맷의 구조가 매우 단순하고 직관적이기 때문에, 개발자들이 코드로 읽고 쓰기 쉬움
- 이 덕분에 유니티(Unity), 언리얼 엔진(Unreal Engine), 3ds Max, Blender 등 대부분의 3D 그래픽 및 게임 엔진과 프로그램에서 기본 포맷으로 강력하게 지원.

## TGA 파일의 주요 기능 및 사양

### 1. 다양한 픽셀 깊이(Bit Depth) 지원

TGA는 이미지의 용도와 필요한 색상 수에 따라 픽셀당 비트 수를 유연하게 지정

- 16비트 (High Color) : RGB 각 5비트씩 사용하고, 남은 1비트를 투명도 오버레이용으로 사용. 용량을 아낄 때 사용.
- 24비트 (True Color) : RGB(레드, 그린, 블루) 각 색상에 8비트(256단계)를 할당하여 약 1,670만 개의 색상을 정밀하게 표현. (투명도 없음)
- 32비트 (True Color + Alpha) : 24비트의 RGB 색상 정보에 8비트의 알파 채널(투명도 256단계)을 추가. 게임 업계에서 가장 흔하게 쓰이는 형태.

### 2. 알파 채널을 활용한 마스킹 (Masking)

- 3D 그래픽에서 캐릭터의 머리카락, 나뭇잎, 쇠창살처럼 복잡한 투명 처리가 필요한 부분에 32비트 TGA 파일을 적용하면, 알파 채널이 마스크(Mask) 역할을 하여 투명한 부분과 불투명한 부분을 정확하게 갈라줌.

### 3.메타데이터 저장

- 1989년에 개정된 TGA 2.0 규격부터는 이미지 데이터 외에도 제작자 이름, 제작 날짜, 이미지 설명, 감마 값, 축소판 이미지(Thumbnail) 등의 확장 메타데이터를 파일 내부에 함께 저장할 수 있는 기능을 제공

## PNG, BMP와의 비교

| 특징 | TGA  | PNG  | BMP  |
| --- | --- | --- | --- |
| 압축 방식 | 비압축 또는 무손실 RLE | 무손실 LZ77 (압축률 높음) | 주로 비압축 |
| 알파 채널 | 매우 안정적이고 명확함 | 지원함  | 지원하나 호환성 떨어짐 |
| 주 사용처 | 게임 엔진 내부 텍스처, 3D 그래픽 | 웹 이미지, 일반 투명 이미지 | 윈도우 시스템 기본 비트맵 |
| 특징 | 채널 처리가 직관적이라 엔진 친화적 | 압축률이 좋아 용량이 작음 | 구조는 단순하나 용량이 너무 큼 |

⇒TGA는 웹 업로드용으로는 용량이 커서 적합하지 않지만, 화질 손실이 없어야 하고, 투명도(알파 채널) 정보를 실시간으로 정확하게 계산해야 하는 게임 및 3D 그래픽 제작 환경에서 핵심 파일 포맷
